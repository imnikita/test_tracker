# Personal Tracker

Личный PWA-трекер тренировок, питания, настроения и фото-прогресса. Парсит твои свободные текстовые записи через Gemini, хранит всё локально в IndexedDB, даёт недельный разбор, сравнивает фото и работает чатом-тренером, который видит твою историю.

## Что внутри

- `index.html` — каркас и UI (6 вкладок: Сегодня, История, Чат, Стата, Фото, Настройки).
- `styles.css` — тёмная тема, мобильная вёрстка.
- `storage.js` — IndexedDB (логи дней + фото-блобы + meta-настройки + история чата) + экспорт/импорт JSON.
- `gemini.js` — четыре режима: парсер (text+photo → структура), аналитик (14 дней → разбор), сравнение фото, чат-тренер с контекстом из логов.
- `app.js` — контроллер, графики на Chart.js, извлечение кадров из видео.
- `manifest.json` + `sw.js` + иконки — обвязка PWA.

## Как запустить

### 1. Получи Gemini API key

Заходишь на https://aistudio.google.com/app/apikey, жмёшь «Create API key», копируешь. На бесплатном тире `gemini-2.0-flash` дают порядка 10–15 запросов в минуту и сотни в день — для одного человека хватит с большим запасом.

### 2. Открыть локально

PWA с service worker'ом не работает по `file://` — нужен HTTP-сервер. Самое простое:

```bash
cd /путь/к/папке
python3 -m http.server 8000
```

Открываешь http://localhost:8000 в браузере. Заходишь во вкладку «⚙» (настройки), вставляешь ключ, сохраняешь. Готово.

### 3. Захостить и поставить на главный экран iPhone

Самый быстрый бесплатный путь:

- **Netlify Drop** (https://app.netlify.com/drop) — перетаскиваешь папку, получаешь HTTPS-ссылку за 30 секунд.
- **GitHub Pages** — `git init`, push в репозиторий, включаешь Pages в настройках.
- **Cloudflare Pages** — то же самое, но быстрее CDN.

Service worker и HTTPS обязательны для установки PWA на iOS. После того как открыл сайт в Safari — «Поделиться → На экран Домой».

## Как этим пользоваться

Открываешь вкладку «Сегодня» и пишешь свободным текстом, например:

> утром овсянка 60г с бананом и ложкой арахисовой пасты. в зале жим лёжа 80×8/7/6, приседания 100×5×3. устал, настроение 6, спал 6.5ч, вес 78.4

Жмёшь «Разобрать и сохранить». ИИ извлекает тренировки, еду, настроение, параметры тела и сохраняет в локальную базу. Если приложить фото — оценит блюдо или просто прикрепит к дню.

Для фото-прогресса — приложи селфи в зеркале на вкладке «Сегодня» вместе с пустым/коротким текстом. Потом во вкладке «Фото» можно будет тапнуть два снимка и нажать «Сравнить через ИИ» — получишь короткое нейтральное описание изменений.

На вкладке «Стата» жмёшь «Получить разбор» — Gemini смотрит на последние 14 дней и пишет, что заметил, плюс 2–3 совета под выбранную в настройках цель.

## Чат с тренером

Во вкладке «Чат» можно задавать вопросы свободно. ИИ получает в контекст твои данные за выбранный период (по умолчанию 14 дней, переключатель сверху — 7/14/30/90/всё) и историю переписки. Примеры вопросов:

- «как мне добрать белок до конца дня?»
- «почему вес стоит на месте?»
- «нормальная ли это форма приседа?» (приложи фото или короткое видео — извлекутся 4 кадра автоматически)
- «что подкорректировать в программе на следующую неделю?»

Видео обрабатываются на устройстве: первые 30 секунд, 4 кадра. Это компромисс — Gemini Flash на видео слабее, чем на картинках, и тратит много токенов; кадры дают приемлемый результат для оценки техники.

Иконка 🗑 рядом с переключателем — очистка всей переписки.

## Бэкап

Во вкладке настроек есть «Экспорт JSON» — выгружает ВСЁ (логи + фото в base64 + настройки кроме API-ключа) в один файл. «Импорт» накатывает обратно. Используй раз в неделю-две, особенно если планируешь чистить кеш браузера.

## Куда расти

- Перенести storage с IndexedDB на Firebase (бесплатный тир) — для синка между устройствами без боли.
- Добавить Capacitor-обёртку — получишь нативные iOS/Android билды из того же кода для App Store.
- Подсчитать справедливые цели по калориям/белку через формулу TDEE на основе вес/рост/возраст/активность.
- Импорт из Apple Health (HealthKit) — работает только в Capacitor-обёртке через плагин.
- Telegram-бот как альтернативный «вход» — пишешь боту, он шлёт запрос в ту же базу.

## Важные оговорки

- Все данные хранятся **в твоём браузере** (IndexedDB). При очистке данных сайта — пропадут. Делай бэкап.
- API-ключ хранится в localStorage **в открытом виде**. Это ОК для личного использования на своём устройстве. Если будешь раздавать другим — нужен бэкенд-прокси, чтобы не светить ключ.
- Калории и БЖУ от ИИ — **оценка** по типичным порциям, а не точные данные с весов. Ошибка ±15–25%, но для отслеживания трендов хватает.
- Сравнение фото даёт **визуальные наблюдения**, а не медицинские оценки. Никаких «процент жира» — модель выдумает.
